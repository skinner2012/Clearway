# Clearway — codebase review (engineering quality)

*Fable 5, 2026-07-15. Scope: code structure, simplicity, and maintenance traps only — methodology and trust design are covered by the separate review (`docs/fable5-review-methodology.md`). Every finding below was verified against the cited lines. Line numbers refer to the tree at `fc128ee`.*

**TL;DR** — the pipeline core (validator, oracle, state machine, store, the math in `eval/stats.py` / `kappa.py` / `confidence.py`) is genuinely clean and should be left alone. The two structural debts are (1) the frozen run/calibration artifacts being the only cross-module shape with no Pydantic model — their dict keys are re-spelled by hand in 4+ consumer files, and (2) a copy-paste layer of infra literals (DB URL, Ollama URL, OTLP endpoint, test probes) that must move together but live apart. The worst traps are silent-failure shaped: a drafter that fails invisibly, a review-approve path that ships the wrong draft, and score-loading code where a renamed key *improves* the score instead of crashing.

---

## 1. What to simplify

### 1.1 Give the run/calibration artifacts a model (the biggest one)

The repo's contract discipline is strict Pydantic (`extra="forbid"`), but the frozen benchmark artifact — the highest-stakes data shape in the system — is a raw dict whose keys are re-spelled by hand in every consumer:

- Produced at `clearway/eval/benchmark_build.py:224-244`; consumed by key-string in `clearway/eval/benchmark.py:66-128` (`cases`, `drafts`, `honest_misses`, `injected`, `tier_b`), `clearway/eval/noise_floor.py:46-69`, `clearway/eval/benchmark_freeze.py:51`.
- Same pattern for calibration rows: produced at `clearway/eval/calibration_build.py:124-147`, consumed in `kappa.py:134-167`, `confidence.py:114-121`, and `calibration_build.py:199-208` — the `"lever" == "natural"` literal alone lives in three files (`kappa.py:166`, `confidence.py:116`, `calibration_build.py:208`).
- The flag-if-any rule `any(is_flag(Conformance(d["conformance"])) for d in c["drafts"])` is written three ways: `benchmark_build.py:216`, `noise_floor.py:51`, and typed at `drafter_score.py:73-76`.
- `tier_b_smoke` returns an untyped dict (`benchmark_tier_b.py:55-71`) that `benchmark.py:117-128` immediately re-parses into `TierBSmoke` with `.get()` defaults.

Simpler shape: one `RunArtifact` / `CalibrationRow` model (or TypedDict) next to the producer, validated on load. Renames then break under mypy, missing keys break at parse time, and traps 2.3 and 2.8 below disappear as a side effect. The three hand-rolled artifact factories in tests (`tests/test_benchmark.py:14-64`, `tests/test_benchmark_freeze.py:13-56`, `tests/test_noise_floor.py:13-51` — already drifted on `config_id`) collapse into one shared factory.

### 1.2 `run()` / `run_set()` are near-duplicates

`clearway/orchestrator/run.py:124-169` vs `172-222`: identical seam-defaulting, oracle construction, `run_id`/timestamp minting, `evaluate(...)` call, review filter, and `save_report`. Make `run(target)` delegate to `run_set([target])`; ~35 lines gone and fixes land once. While there: the review filter at `run.py:166` and `run.py:219` loads **every review across all runs** and filters in Python — the store protocol scopes everything else by `run_id`; add the filter to `load_reviews` (`store.py:81`).

### 1.3 Deduplicate the infra literals

Each of these is a set of literals that must move together but is spelled in multiple files:

- **Ollama base URL + env var** — `corpus/embed.py:28,60`, `llm/local.py:20,28`, `eval/benchmark_build.py:60`. One `ollama_base_url()` in `clearway/llm/`.
- **OTLP endpoint + OTel `Resource`** — duplicated between `observability/metrics.py:27,97-98,107` and `observability/tracing.py:25,36-37,39`. The `service.instance.id` is load-bearing (the comment at `metrics.py:104-106` explains why) yet exists twice.
- **Postgres URL** — `postgresql://clearway:clearway@localhost:5432/clearway` appears twice in source (`corpus/store.py:17`, `orchestrator/store.py:35`) and 12+ times in tests.
- **Test gate probes** — `_ollama_up()` / `_postgres_up()` / skip markers are verbatim copies across `tests/test_corpus.py:158-179`, `tests/test_retriever.py:142-163`, `tests/test_mcp_server.py:177-198`, `tests/test_drafter.py:153-161`, `tests/test_cli.py:43`, `tests/test_validator.py:150`, `tests/test_orchestrator_store.py:244`. There is no `tests/conftest.py`; one would absorb all of this plus the identical try/finally DELETE cleanup blocks (`test_corpus.py:200-205`, `test_retriever.py:186-191`, `test_mcp_server.py:221-226`).
- **Eval layout paths** — `benchmark_freeze.py:24` and `noise_floor_build.py:23` import underscore-private `_REPORTS_DIR`/`_RUNS_DIR` from `benchmark_build`; `_NOISE_FLOOR` is independently re-defined in both (`benchmark_freeze.py:27`, `noise_floor_build.py:26`); `acceptance_snapshot.py:16` re-derives the scorecard path instead of reusing `benchmark_freeze._SCORECARD`; `_FIXTURES` is re-derived in five files. One small shared paths module fixes all of it.

### 1.4 Make the cross-module seams public

Three places import another module's `_underscore` internals; each is a rename away from breakage with no deprecation surface:

- `clearway/eval/calibration_build.py:35-36` imports `_assemble`, `_LLMDraft`, `_system_prompt` from `clearway.drafter.llm`. Give the drafter an explicit entry point for supplying a custom user prompt.
- `clearway/cli.py:88-94` imports `run._default_store`; both CLI test fixtures monkeypatch the same private names (`tests/test_cli.py:36-38`, `121-125` — themselves duplicated; `shared_spine` subsumes `offline_spine`).
- The `_REPORTS_DIR`/`_RUNS_DIR` imports above.

### 1.5 Shared LLM-call scaffolding

- The call→validate→retry loop is duplicated between `drafter/llm.py:72-78` and `judge/judge.py:115-120`. A shared `complete_validated(...) -> tuple[Model, LLMUsage] | None` keeps their deliberately different failure policies (fallback vs raise) while deleting the loop.
- `_usage_from` is structurally identical in `llm/local.py:50-63` and `llm/cloud.py:65-78`; collapsing them forces resolution of the `cost_usd` inconsistency (trap 2.12).

### 1.6 Smaller cuts

- **Empty subclasses + TypeVar**: `AxeViolation`/`AxeIncomplete`/`AxePass` (`schemas/models.py:125-139`) are docstring-only subclasses nothing dispatches on — bucket identity is already carried by the `ScanResult` list and `Finding.source_bucket`; the normalizer switches on the `(bucket, rules)` tuple (`normalize.py:110-114`). One `AxeRuleResult` in all three lists deletes the `_RuleResultT` TypeVar (`scanner/scan.py:30`). Stable, so low priority.
- **Dead code**: `act_gold.load_act_gold_pairs` (`act_gold.py:140-165`) has zero callers; `fp_k` at `drafter_score.py:160` is assigned and never used; `edit_distance.py:50` re-filters a list already filtered at line 47.
- **Grep-hostile twin names**: `kappa.build_report` → `CalibrationReport` (`kappa.py:184`) vs `benchmark.build_report` → `BenchmarkReport` (`benchmark.py:153`). Rename one.
- **`_step()` takes 10 parameters** (`machine.py:314-324`), six constant across all three call sites in `_process_finding` (`machine.py:155-210`). A per-run context object reduces each call to the four that vary.
- **Two gauge-registration styles in `metrics.py`**: the four legacy gauges use module globals + `assert is not None` (`metrics.py:83-86, 116-131, 257-266`) while the newer gauges use the name→description dict pattern (`metrics.py:39-80, 132-135`). Fold the legacy four into the dict.

## 2. Traps and danger zones

### High

**2.1 A totally-failed drafter is invisible on every telemetry surface.** When the model never returns parseable JSON, `draft_with_usage` returns a fallback row (`drafter/llm.py:72-79`, `139-149`) instead of raising. Consequences chain: `pipeline_failures` never increments (`machine.py:374` fires only on a raised exception); `pipeline_step_retries` never increments (the parse-retry lives *inside* the drafter, below the orchestrator's attempt counter); and the fallback carries `LLMUsage()` — all `None` — so `record_llm_call` records nothing (`operational.py:104-109` skips every `None` field), erasing even the failed calls' token spend and latency. Nothing downstream checks `confidence == 0.0` or `_FALLBACK_CONFIDENCE` (grep: only the definition and a docstring). Failure scenario: the local model starts emitting markdown for every finding; the dashboard shows zero failures, zero retries, zero LLM traffic, and a run full of `does_not_support` rows. The fallback-not-crash policy is fine — but the fallback path needs to emit a counter.

**2.2 Edit-then-approve silently ships the original draft.** `_review_approve_cmd` preserves `edited_draft` while setting status APPROVED (`cli.py:177`), but the resume gate honors the edit only when `status is ReviewStatus.EDITED` (`machine.py:309`). A reviewer who runs `review edit` then `review approve` (a natural "finalize my edit" gesture) ships the **original** draft while `review show` displays the edit as attached — record and output disagree, no warning. No test covers the edit-then-approve sequence (`test_cli.py` tests each in isolation). Either `approve` should refuse/warn when an edit exists, or the gate should honor any non-`None` `edited_draft`.

**2.3 Missing or renamed artifact keys silently *improve* benchmark scores.** `benchmark.py:87` reads `artifact.get("honest_misses", [])` — if the key is absent (renamed in the builder, truncated file), the failed honest-misses drop out of the recall denominator, which is exactly the overstatement `drafter_score.py:12-14` warns about in bold. `benchmark.py:109-114` reads `artifact.get("injected", {}).get(key, [])` — a producer-side rename of `"conformance_flip"`/`"sc_swap"` (written at `benchmark_build.py:242`) turns both detection rates into silent n=0 "no data". `noise_floor.py:53,69` repeat the `honest_misses` default. The pure-test suite can't catch this: tests hand-build artifacts with the consumer's spelling (`test_benchmark.py:127-143`). "Optional section" and "corrupt key" are indistinguishable to this code — the artifact model (§1.1) with explicit-optional fields separates them.

**2.4 `judge_version` promises to track prompt edits but hashes half the prompt.** `_RUBRIC_HASH` covers only `_RUBRIC_SYSTEM` (`judge/judge.py:58-60` — the comment says a hash exists so "a changed rubric [can't] masquerade as the calibrated one"), but the user-prompt template in `_judge_user_prompt` (`judge.py:137-149`) — its wording, field framing, and field selection — is unhashed. Reword or reorder the template and every `JudgeResult` still carries the old `judge_version`; the recalibration trigger never fires. One-line fix: hash the system rubric plus the template skeleton (e.g. the template applied to a fixed sentinel input).

**2.5 The pinned axe-core version is metadata nothing verifies.** `AXE_VERSION = "4.12.1"` (`scanner/scan.py:32-34`) claims to match `vendor/axe.min.js`, but no code or test ever reads `axe.version` from the injected engine (grep: zero hits). The test that looks like coverage, `tests/test_scanner.py:52`, asserts the constant against itself. Swap the vendored file without bumping the constant and every `ScanResult.tool_version` and the frozen benchmark's `axe_core_version` (`models.py:896`) records the wrong engine, silently, forever — in a repo whose premise is pinned reproducibility. One-line fix: assert `page.evaluate("axe.version") == AXE_VERSION` inside `scan()`.

### Medium

**2.6 Fresh multi-page runs print spurious "resuming run" notices — and a test enshrines it.** `execute()` fires the resume hook whenever any steps exist under the `run_id` (`machine.py:87, 94`), but `run_set` calls `execute()` once per page under one shared `run_id` (`run.py:200-211`) with the hook always wired (`cli.py:319-320`). A fresh 3-page `clearway eval` prints "resuming run …: 0/3 findings already complete" for pages 2 and 3; the contract comment "Called once, only when resuming" (`machine.py:64-65`) is false on this path. `test_on_resume_hook_scopes_counts_to_the_current_batch...` (`tests/test_machine.py:249-259`) asserts the spurious firing as correct. Related: "done" counts VALIDATE-DONE (`machine.py:99-103`), so a real resume with pending reviews prints "N/N already complete, nothing left to do" while findings are still withheld.

**2.7 `RunStatus.DONE` is unconditional and flaps per page.** Every `execute()` ends by stamping DONE (`machine.py:128`) regardless of FAILED steps, and `run_set` cycles RUNNING→DONE once per page. Kill a multi-page eval between pages and `load_run()` reads DONE with pages never scanned. A test pins DONE-after-FAILED as intended (`tests/test_mcp_retrieve.py:232-233`), so this is a design choice — but the name overstates what DONE guarantees.

**2.8 Tier-B's `clean_flags.get(id, flagged_noisy)` fabricates "no change under noise".** `benchmark_build.py:131`: if a focal's `act_testcase_id` is missing from the clean-run map (typo in the noisy-pages manifest, or the counterpart becoming an honest miss after an axe bump — the map at `benchmark_build.py:215-217` is built from minting cases only), the clean verdict silently defaults to the noisy verdict, forcing the clean-vs-noisy delta — the one number Tier B measures — to zero for that page.

**2.9 The crash-recovery checkpoint is not crash-safe.** `_write_partial` overwrites `run.partial.json` in place after every case (`benchmark_build.py:68-71`); a crash mid-write leaves truncated JSON, and the next start dies in `_read_partial`'s bare `json.loads` (`benchmark_build.py:65`) — losing multi-hour state in exactly the abrupt-death scenario the checkpoint exists for. Write-temp-then-`os.replace` is a two-line fix.

**2.10 `eval` ↔ `orchestrator` package cycle (boundary violation).** `orchestrator/run.py:21` imports `clearway.eval`; `eval/confidence_build.py:25` imports `clearway.orchestrator.run` — `ARCHITECTURE.md` §6 says nothing may depend on `orchestrator/` except cli/api. The cycle is dormant only because `eval/__init__.py:3-8` happens not to export `confidence_build`; re-exporting it (the natural move) makes `import clearway.orchestrator.run` a startup circular-import error. Every other module honors the boundary (verified by grep). Two softer §6 divergences: `corpus/wcag.py:24` and `validator/check.py:13` import from `oracle/`, which the table lists as dependency-free.

**2.11 Observability has two contradictory before-setup policies, and re-setup silently kills pipeline metrics.** The trust-metric recorders auto-setup on first use — `record_rate` (`metrics.py:140-141`), `record_eval_report` (`155`), `record_calibration` (`185-186`), `record_benchmark` (`248-249`) — constructing an OTLP exporter to `localhost:4318` and claiming the process-global meter provider as a side effect of "record a number"; the operational recorders are documented no-ops before setup (`operational.py:101-102`). Worse: `shutdown()` nulls `_provider` inviting re-setup (`metrics.py:255-268`), but OTel's global setter is once-per-process (`metrics.py:111`), so after shutdown→setup the trust gauges work while `setup_operational_metrics()` binds to the dead global provider (`operational.py:75`) and every `record_llm_call`/`record_step` is silently dropped. Pick the no-op policy everywhere and set up explicitly at the CLI entry point.

**2.12 Assorted silent-failure spots.** (a) `scan()` never checks the `page.goto` response status (`scan.py:73`) — a 404/500 page scans "clean"; `results.get(..., [])` at `scan.py:84-88` also degrades a malformed axe payload to "no findings". Mostly bites the live-scan path. (b) `llm/local.py:61-62` swallows a pricing failure to `0.0` while `llm/cloud.py:76-77` swallows the same to `None` — local conflates "unknown" with "free", against `LLMUsage`'s own contract (`client.py:20-23`). (c) `confidence_build.py:50-51` silently drops an oracle-checked draft with `confidence is None` from the calibration curve — the one deviation from the eval family's raise-don't-fabricate discipline (`confidence.py:85-88`, `stats.py:69-70`). (d) The `QUALITY_REVIEW_RULES[rule.rule_id]` lookup (`normalize.py:76`) is guarded only by a `continue` in its *caller* (`normalize.py:117-118`) — a future caller gets a bare `KeyError`.

### Low

**2.13 Docstrings that assert the opposite of the code.** `judge_score.py:17-18` and `108-110` say the conformance-flip's rationale "was regenerated to argue the flip"; `benchmark_inject.py:10-15, 36-41, 60-63` state and implement the opposite (pure `model_copy`, no LLM, deliberately). The stale text sits on the measurement surface a reviewer reads first. Similar staleness: `_to_rule_result`'s docstring (`scan.py:49-50`) omits that it now also maps `passes`; `_ECE_EXEMPT_REASON` (`drafter_score.py:38-41`, attached unconditionally at `:201`) hard-codes "single populated confidence bin", true of today's data but unasserted — a future two-bin run ships a false exemption reason.

**2.14 Schema invariants that live only in prose.** `EvalMetrics`' partition invariant (`models.py:435`), `MetricCI` ordering (`models.py:680-696`), and `ConfidenceBin` bounds (`models.py:612-629`) are promised in descriptions but never validated — each is a one-line `model_validator`. The tests "covering" them assert arithmetic on their own fixtures (`test_schemas.py:184-193`; the `_ci()` helper at `:468-469` even clamps its inputs legal). Related: provenance `source` fields are free strings duplicated as prose in an otherwise enum-strict file (`models.py:227, 251, 525, 575-580`; written as a bare literal at `oracle/axe.py:69`) — a future `"expert_gold"` vs `"expert-gold"` typo validates fine.

**2.15 The in-memory store diverges from Postgres in the untested gap.** Three observable differences, all shielded only because Pg tests skip when the stack is down (`test_orchestrator_store.py:250`): `save_step(result_json=None)` keeps the old result in memory but nulls it in Pg (`store.py:122-126` vs `233-235`); `save_run` upsert preserves `created_at` in Pg but replaces the row in memory (`store.py:213` vs `116-117`); `load_reviews` orders by `created_at` in Pg vs insertion order in memory (`store.py:313` vs `140-141`) — and all reviews in one run share a timestamp (`machine.py:295-296`), so `test_run_set_edit_reflow...`'s `edited, approved = pending` unpack (`test_orchestrator.py:195`) would flake on Pg. Same shape in the corpus fake: `InMemoryCorpusStore.upsert` partially commits before raising (`corpus/store.py:87-92`) where `PgCorpusStore` validates the whole batch first (`store.py:174-188`). Also: `Retriever._sc_meta_map` caches an empty meta map forever (`rag.py:60-65`).

**2.16 Hollow asserts (individually harmless, inflate apparent coverage).** `test_retriever.py:95` (`_DEFAULT_K == 5` — constant vs literal); `test_corpus.py:100` (`build_corpus_version` can't be changed by `upsert_sc_meta`); `test_edit_distance.py:83-86` (re-implements the `SequenceMatcher` formula and asserts the function equals it — the only formula-reimplementing test in the suite); `test_schemas.py:156-157` (evaluates the same f-string the `EvidenceQuery` docstring hardcodes — which itself documents the *retriever's* private query construction in the schema module, `models.py:197-202`); `test_orchestrator_store.py:118-119` (`runtime_checkable` `isinstance` checks method names only). Also `cli.py:153`: a failing/missing `$EDITOR` escapes as a raw traceback.

## 3. Already good enough — do not touch

- **`validator/check.py`** — the best module in the repo. The L0/L1 decision table is three tiny pure functions (`check.py:25-40`), ground truth flows only through the `Oracle` protocol (`check.py:55-62`), and `tests/test_validator.py` covers all four verdict paths including real-but-wrong-SC (`test_validator.py:87-92`).
- **`oracle/`** — tight tag regex that rejects level/category tags (`axe.py:18`), defensive `VALID_SC_IDS` filter (`axe.py:35-36`), bucket allowlist in the right place (`axe.py:61-62`); `wcag.py` is a pure data module whose 86-entry table matches its own docstring and pin test.
- **The state machine's failure discipline** (`machine.py:342-389`) — the broad `except` is deliberate and loud: every attempt is a span event, exhaustion records the exception and checkpoints FAILED, and the `is None` sentinel correctly distinguishes falsy-but-valid results (`[]` citations) from failure (`machine.py:167, 194, 212`). The replay path and the `usage_box` honest-`None`-on-replay design (`machine.py:173-198`) are exactly right.
- **`orchestrator/mcp_retrieve.py`** — raises on `isError` and on missing `structuredContent` (`mcp_retrieve.py:58-68`) instead of returning `[]`; the connect-per-call rationale (`:21-25`) is sound.
- **`PgOrchestratorStore`** — parameterized queries throughout; natural-key upserts match the machine's idempotency assumptions; DDL matches the read/write columns.
- **The eval math**: `stats.py` (Wilson with bracket snap `:81-85`; `COLLAPSE_RULE` generated *from* the flag sets so the audit string can't drift, `:40-48`), `kappa.py` (negative κ preserved, no-variance edge documented, `:74-88`), `confidence.py` (half-open bins with closed top, raise-on-empty), `benchmark_inject.py` (pure transforms, decoy exhaustion raises). Tests assert hand-computed constants, not re-implemented formulas.
- **The drift guards fail loud**: the gold-set finding-count assertion (`act_gold.py:148-152`), the 68-file checksum pin (`test_act_gold.py:34-47`), `noisy_pages._assert_composition` (`noisy_pages.py:186-195`), and `benchmark_freeze._assert_drafter_deterministic` (`benchmark_freeze.py:31-43`, tested at `test_benchmark_freeze.py:78-83`).
- **The finding-id scheme** (`normalize.py:48-62`) with its independently-recomputed digest test (`test_normalizer.py:120-125`) and the frame-path collision test (`test_normalizer.py:102-107`) — genuinely non-tautological pins.
- **`llm/client.py`** — `Completion` carrying usage prevents telemetry-discarded-at-the-boundary by construction. **`observability/operational.py`** — source imports semconv names from library constants while tests assert the literal wire names (`test_observability_operational.py:101-110, 154-168`): exactly the right split.
- **`corpus/wcag.py`, `corpus/ingest.py`, `retriever/rag.py`, `mcp_server/server.py`** — small, single-purpose; `zip(strict=True)` guards the embed/chunk pairing; retrieve/retrieve_query genuinely funnel into one core with parity tested structurally and over the wire (`test_retriever.py:119-129`, `test_mcp_server.py:105-121`); the traceparent extraction degrades to a root span and both branches are asserted against real finished spans.
- **The test suites as a whole** — fakes sit *behind* the code under test rather than replacing it; the HITL reflow tests drive the full withhold→approve/edit/reject→resume loop against real metrics (`test_orchestrator.py:141-208`, `test_cli.py:147-194`); replay tests count actual seam invocations; the span-propagation test asserts the real parent chain (`test_mcp_retrieve.py:369-375`). The hollow asserts in §2.16 are the exceptions, not the rule.
